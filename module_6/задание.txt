Предположим, что мы пользуемся приложением Яндекс.Такси на мобильном телефоне. 
Также есть веб-версия (https://taxi.yandex.ru/). В отличие от мобильного приложения, в браузере нельзя посмотреть историю своих поездок.
Бизнес попросил ввести историю поездок и в веб-версию в том числе. 
Итак, появляется верхнеуровневое бизнес-требование «Должна быть история поездок в веб-версии».

Задание:
1. Составить User Story Mapping в Trello на это верхнеуровневое требование, сделать доску публичной и выслать ссылку на доску в качестве ответа.
2. Составить Example Mapping в виде Mind Map на декомпозированные в п.1 требования. 
Использовать для выполнения инструмент xmind. Маинд-карту конвертируйте в PDF, загрузите получившийся файл на любой файлообменник и предоставить ссылку.

Требования к выполнению:
1. Тема для User Story Mapping — «История поездок в веб-версии Яндекс.Такси». 
К ней нужно придумать одну крупную User Story (эпик) и 3 мелких User Story. Количество задач для каждой User Story — 2.
2. Для каждой User Story выработать 2 критерия приемки в рамках Example Mapping.
3. Для каждого критерия приемки разработать тест-кейс в рамках Example Mapping.
4. Если будут возникать вопросы к реализации User Story, можно составить к ним список вопросов.
